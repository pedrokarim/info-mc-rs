mc-protocol

Custom Minecraft protocol implementation with zero third-party protocol dependencies. This crate handles all low-level communication with Minecraft servers, including Java Edition Server List Ping (SLP), Bedrock Edition pinging, and the legacy Query protocol.

StandaloneYes

Key Types

The protocol types handle Minecraft's custom variable-length encoding:

/// Variable-length integer (1-5 bytes), used extensively in the protocol.
pub struct VarInt(pub i32);

/// Variable-length long (1-10 bytes), used for entity IDs and timestamps.
pub struct VarLong(pub i64);

/// Length-prefixed UTF-8 string with a maximum byte length.
pub struct McString(pub String);

/// A raw protocol packet with ID and payload.
pub struct McPacket {
    pub id: VarInt,
    pub data: Vec<u8>,
}

Key Functions

ping_java

Performs a full Server List Ping against a Java Edition server. Returns the server's MOTD, player count, version, favicon, and latency.

use mc_protocol::slp::ping_java;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let response = ping_java("mc.hypixel.net", 25565).await?;

    println!("Version: {}", response.version.name);
    println!("Players: {}/{}", response.players.online, response.players.max);
    println!("Latency: {}ms", response.latency_ms);
    println!("MOTD: {:?}", response.description);

    Ok(())
}

ping_bedrock

Sends an unconnected ping to a Bedrock Edition server using the RakNet protocol and parses the response string.

use mc_protocol::bedrock::ping_bedrock;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let response = ping_bedrock("play.example.com", 19132).await?;

    println!("Edition: {}", response.edition);
    println!("MOTD: {}", response.motd);
    println!("Players: {}/{}", response.players_online, response.players_max);
    println!("Game mode: {}", response.game_mode);

    Ok(())
}

query_server

Uses the UDP Query protocol to retrieve extended server information, including the player list and installed plugins. Requires enable-query=true in the server's server.properties.

use mc_protocol::query::query_server;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let stats = query_server("mc.example.com", 25565).await?;

    println!("Map: {}", stats.map);
    println!("Plugins: {:?}", stats.plugins);
    println!("Online players: {:?}", stats.players);

    Ok(())
}

resolve_srv

Resolves _minecraft._tcp.<domain> SRV records to find the actual host and port of a Minecraft server.

use mc_protocol::dns::resolve_srv;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    if let Some((host, port)) = resolve_srv("example.com").await? {
        println!("Resolved to {}:{}", host, port);
    } else {
        println!("No SRV record found, using default port 25565");
    }

    Ok(())
}

Design Notes

  • The protocol implementation is fully hand-written — no code generation or third-party Minecraft protocol libraries.
  • VarInt/VarLong encoding follows the wiki.vg specification exactly.
  • All network I/O is async via tokio, with configurable timeouts to avoid hanging on unresponsive servers.
  • SRV resolution uses hickory-resolver (formerly trust-dns) for pure-Rust DNS lookups.
