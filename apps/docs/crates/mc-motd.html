mc-motd

Minecraft MOTD (Message of the Day) parser and renderer. Handles both legacy section-sign (§) color codes and modern Chat Component JSON format, converting them to HTML or plain text.

StandaloneYes
Pathcrates/mc-motd

FilePurpose

types.rsCore types: MotdNode, McColor, MotdStyle

Key Types

/// A node in the parsed MOTD tree.
pub struct MotdNode {
    pub text: String,
    pub style: MotdStyle,
    pub children: Vec<MotdNode>,
}

/// Minecraft color codes (0-9, a-f).
pub enum McColor {
    Black,       // §0
    DarkBlue,    // §1
    DarkGreen,   // §2
    DarkAqua,    // §3
    DarkRed,     // §4
    DarkPurple,  // §5
    Gold,        // §6
    Gray,        // §7
    DarkGray,    // §8
    Blue,        // §9
    Green,       // §a
    Aqua,        // §b
    Red,         // §c
    LightPurple, // §d
    Yellow,      // §e
    White,       // §f
    Hex(String), // Custom hex color (1.16+)
}

/// Text styling flags.
pub struct MotdStyle {
    pub color: Option<McColor>,
    pub bold: bool,
    pub italic: bool,
    pub underlined: bool,
    pub strikethrough: bool,
    pub obfuscated: bool,
}

Exported Functions

parse_motd

Parses a raw MOTD string (either legacy format or JSON) into a tree of MotdNode values:

use mc_motd::{parse_motd, to_html, to_plain_text};

// Legacy format with section signs
let nodes = parse_motd("§aWelcome to §b§lMyServer§r§7!");

// Chat Component JSON format
let json_motd = r#"{"text":"","extra":[{"text":"Welcome","color":"green"},{"text":" to ","color":"gray"},{"text":"MyServer","color":"aqua","bold":true}]}"#;
let nodes = parse_motd(json_motd);

to_html

Renders a parsed MOTD to HTML with inline CSS for colors and styles:

use mc_motd::{parse_motd, to_html};

let nodes = parse_motd("§aWelcome to §b§lMyServer");
let html = to_html(&nodes);
// Output: <span style="color:#55FF55">Welcome to </span>
//         <span style="color:#55FFFF;font-weight:bold">MyServer</span>
println!("{}", html);

to_plain_text

Strips all formatting codes and returns the raw text content:

use mc_motd::{parse_motd, to_plain_text};

let nodes = parse_motd("§aWelcome to §b§lMyServer");
let plain = to_plain_text(&nodes);
assert_eq!(plain, "Welcome to MyServer");

render_motd

Convenience function that auto-detects the format and directly returns HTML:

use mc_motd::render_motd;

// Works with both legacy and JSON formats
let html = render_motd("§6§lHypixel Network §7[1.8-1.20]");
println!("{}", html);

Format Detection

The parser automatically detects whether the input is:

  • Legacy format — Contains § (section sign) followed by color/formatting codes
  • Chat Component JSON — Starts with { or [, parsed as a structured JSON tree with text, extra, color, bold, etc. fields
  • Plain text — No formatting detected, returned as-is

Design Notes

  • The parser handles nested Chat Components recursively, supporting deeply nested extra arrays as used by servers like Hypixel.
  • Hex color support (§x§R§R§G§G§B§B) covers the custom color format introduced in Minecraft 1.16.
  • The HTML renderer sanitizes text content to prevent XSS when displaying server MOTDs in browsers.
  • Obfuscated text (§k) is rendered with a CSS class that can be animated client-side.
