mc-mojang

HTTP client for Mojang's official API. Provides typed access to player profile lookups, UUID resolution, skin URLs, cape information, and skin model detection.

StandaloneYes
Pathcrates/mc-mojang

Source Structure

FilePurpose
client.rsHTTP client wrapping Mojang API calls
types.rsResponse types: PlayerProfile, SkinInfo, CapeInfo, SkinModel
error.rsMojang-specific error types (rate limited, not found, etc.)

Key Types

/// A player's full profile from Mojang's session server.
pub struct PlayerProfile {
    pub id: String,          // UUID (no dashes)
    pub name: String,        // Current username
    pub skin: Option<SkinInfo>,
    pub cape: Option<CapeInfo>,
}

/// Skin texture information.
pub struct SkinInfo {
    pub url: String,         // URL to the skin PNG on Mojang's CDN
    pub model: SkinModel,    // Classic (4px arms) or Slim (3px arms)
}

/// Cape texture information.
pub struct CapeInfo {
    pub url: String,         // URL to the cape PNG
}

/// Skin arm model variant.
pub enum SkinModel {
    Classic,   // Steve-style 4px wide arms
    Slim,      // Alex-style 3px wide arms
}

Usage Examples

Look up a player by username

use mc_mojang::client::MojangClient;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = MojangClient::new();

    // Resolve username to UUID
    let uuid = client.username_to_uuid("Notch").await?;
    println!("UUID: {}", uuid);

    // Fetch full profile with skin/cape info
    let profile = client.get_profile(&uuid).await?;
    println!("Name: {}", profile.name);

    if let Some(skin) = &profile.skin {
        println!("Skin URL: {}", skin.url);
        println!("Model: {:?}", skin.model);
    }

    if let Some(cape) = &profile.cape {
        println!("Cape URL: {}", cape.url);
    }

    Ok(())
}

Batch UUID lookups

use mc_mojang::client::MojangClient;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let client = MojangClient::new();

    let usernames = vec!["Notch", "jeb_", "Dinnerbone"];
    for name in usernames {
        match client.username_to_uuid(name).await {
            Ok(uuid) => println!("{} -> {}", name, uuid),
            Err(e) => eprintln!("{}: {}", name, e),
        }
    }

    Ok(())
}

Design Notes

  • The client decodes the base64-encoded textures property from Mojang's session server to extract skin/cape URLs and model type.
  • Mojang's API is rate-limited; callers should wrap this crate with mc-cache in production to avoid hitting limits.
  • All API responses are strongly typed with serde deserialization, providing compile-time safety.
  • The SkinModel enum is derived from the presence of a "metadata": {"model": "slim"} field in the textures payload.
