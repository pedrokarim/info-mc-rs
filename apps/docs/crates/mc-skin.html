mc-skin

Skin downloading, caching, and 2D isometric rendering. This crate handles fetching player skins from Mojang's CDN, detecting the arm model (slim vs. classic), and generating 2D rendered views including head, bust, full body, and face extractions.

StandaloneNo (depends on mc-mojang)
Pathcrates/mc-skin

Source Structure

FilePurpose
fetch.rsDownloads skin PNGs from Mojang's texture CDN
model.rsSkin model detection (slim vs. classic) and skin region mapping
render2d.rsIsometric 2D rendering engine for head, bust, full body, face views
cache.rsLocal PNG file caching for downloaded skins

Features

  • Download from Mojang CDN — Fetches the 64x64 skin PNG via the texture URL from mc-mojang
  • Slim/Classic detection — Automatically detects whether the skin uses 3px (Alex) or 4px (Steve) arm widths
  • Isometric 2D renders — Generates pixel-art style rendered views:
    • Face — Front face of the head (8x8 pixels, scaled up)
    • Head — Isometric 3D-looking head with hat overlay
    • Bust — Head + upper body
    • Full — Complete player body with all overlay layers
  • Local PNG caching — Caches downloaded skin files to disk to avoid repeated Mojang API calls

Usage Examples

Fetch and render a player's skin

use mc_skin::fetch::fetch_skin;
use mc_skin::render2d::{render_face, render_head, render_bust, render_full};

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    // Fetch the skin image from Mojang's CDN
    let skin_url = "http://textures.minecraft.net/texture/abc123...";
    let skin_image = fetch_skin(skin_url).await?;

    // Render different views
    let face = render_face(&skin_image, 128);      // 128px face
    let head = render_head(&skin_image, 160);      // 160px isometric head
    let bust = render_bust(&skin_image, 256);      // 256px bust
    let full = render_full(&skin_image, 300);      // 300px full body

    // Save to PNG
    face.save("face.png")?;
    head.save("head.png")?;
    bust.save("bust.png")?;
    full.save("full.png")?;

    Ok(())
}

Detect skin model

use mc_skin::model::SkinModel;
use image::open;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let skin = open("skin.png")?.to_rgba8();
    let model = SkinModel::detect(&skin);

    match model {
        SkinModel::Classic => println!("Steve-style (4px arms)"),
        SkinModel::Slim => println!("Alex-style (3px arms)"),
    }

    Ok(())
}

Using the skin cache

use mc_skin::cache::SkinCache;

#[tokio::main]
async fn main() -> Result<(), Box<dyn std::error::Error>> {
    let cache = SkinCache::new("./skin_cache");

    // Returns cached version if available, otherwise downloads
    let skin = cache.get_or_fetch("player_uuid", "https://textures.minecraft.net/...").await?;

    println!("Skin dimensions: {}x{}", skin.width(), skin.height());

    Ok(())
}

Skin Texture Layout

Minecraft skins are 64x64 PNG images with a specific pixel layout. The crate maps these regions for rendering:

64x64 Skin Texture Map
+--------+--------+--------+--------+
|        |  Head  | Head   |        |
|        |  Top   | Front  |        |
| (0,0)  | (8,0)  | (8,8)  |        |
+--------+--------+--------+--------+
|  Leg   |  Body  | Body   | Arm    |
|  Right |  Front | Right  | Right  |
+--------+--------+--------+--------+
| (Bottom half: overlay layers)      |
| Hat, Jacket, Sleeves, Pants        |
+------------------------------------+

Design Notes

  • The 2D renderer uses nearest-neighbor interpolation to preserve the pixel-art aesthetic when scaling up the tiny skin textures.
  • Overlay layers (hat, jacket, sleeves, pants) are composited on top of the base layers with alpha blending.
  • The isometric head renderer applies a pseudo-3D transformation by skewing the top and side faces of the head.
  • All rendering is done CPU-side using the image crate — for GPU-accelerated 3D rendering, see mc-render3d.
