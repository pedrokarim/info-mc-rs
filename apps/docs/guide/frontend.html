Frontend Development

The MCInfo frontend is a SvelteKit application located in apps/web. It provides a rich, responsive interface for browsing Minecraft servers and player profiles, complete with an interactive 3D skin viewer powered by Three.js.

Stack Overview

Technology Purpose
SvelteKit Full-stack framework with SSR, routing, and API proxying
Three.js Interactive 3D skin viewer in the browser
Bun Package manager and development runtime
Vite Build tool (via SvelteKit) 
# Development
cd apps/web
bun install
bun run dev

# Production build
bun run build
bun run preview

Key Components

Hero Section

The landing page features a hero section with an animated gradient background, a search bar for server/player lookup, and floating Minecraft-themed decorative elements.

Server Card

Displays server information in a compact card format: favicon, MOTD (rendered with Minecraft formatting), player count, version, and latency indicator. Cards support hover animations and link to the full server detail page.

Player Profile

Shows player information including username, UUID, skin preview, name history, and associated capes. The profile page includes both a 2D face render and an interactive 3D skin viewer.

MOTD Renderer

Renders Minecraft MOTDs with full support for color codes, formatting, and both legacy (§) and JSON chat component formats. The renderer produces styled HTML that accurately matches in-game appearance.

Skin 3D Viewer

An interactive Three.js-based viewer that renders the player's skin as a 3D model. Supports orbital rotation (drag to rotate), walking animation toggle, and zoom. Falls back to a static server-rendered image when WebGL is unavailable.

Responsive Breakpoints

The layout adapts to different screen sizes using the following breakpoints:

Breakpoint Width Behavior
Mobile < 640px Single column, hamburger menu, stacked cards
Tablet 640px – 1024px Two-column grid, collapsible sidebar
Desktop 1024px – 1440px Full layout with sidebar, three-column card grid
Wide > 1440px Max-width container, four-column card grid 
/* Responsive media queries */
@media (max-width: 640px)  { /* Mobile styles */ }
@media (max-width: 1024px) { /* Tablet styles */ }
@media (min-width: 1440px) { /* Wide screen styles */ }

Animation Effects

The frontend uses subtle animations to enhance the user experience:

  • Page transitions — fade and slide transitions between routes using SvelteKit's built-in transition system
  • Card hover — scale transform (1.02x) with elevated shadow on hover, 200ms ease transition
  • Skeleton loading — animated gradient pulse effect for loading states
  • MOTD obfuscation — rapid character randomization matching in-game behavior at 20 ticks/second
  • 3D viewer rotation — smooth orbital rotation with momentum/inertia on mouse release
  • Scroll reveal — elements fade in as they enter the viewport using IntersectionObserver

SEO and OpenGraph

MCInfo uses SvelteKit's server-side rendering to ensure all pages are fully indexed by search engines. Each page includes comprehensive meta tags:

# Server page: /server/play.example.com
<title>play.example.com - Minecraft Server Status | MCInfo</title>
<meta name="description" content="..." />
<meta property="og:title" content="play.example.com" />
<meta property="og:description" content="42/100 players online..." />
<meta property="og:image" content="/api/server/play.example.com/icon" />

# Player page: /player/Notch
<title>Notch - Minecraft Player Profile | MCInfo</title>
<meta property="og:image" content="/api/player/Notch/render3d" />

The OpenGraph images are dynamically generated by the API using the server-side rendering pipeline, ensuring rich link previews on Discord, Twitter, and other platforms.
