Getting Started

Welcome to the MCInfo-RS documentation. This guide will walk you through setting up the project locally and understanding its overall structure.

Prerequisites

Before you begin, make sure you have the following tools installed on your system:

Tool Version Purpose
Rust Stable (latest) Backend API, protocol implementation, skin rendering
Bun 1.x Frontend package manager and runtime

Install Rust via rustup:

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

Install Bun via the official installer:

curl -fsSL https://bun.sh/install | bash

Backend Setup

The backend is a Rust workspace powered by Axum. To start the API server:

# Clone the repository
git clone https://github.com/your-org/info-mc-rs.git
cd info-mc-rs

# Run the API server (port 3001)
cargo run -p mc-api

The API server starts on port 3001 by default. You should see output similar to:

INFO  mc_api > Listening on 0.0.0.0:3001

Note: The first build may take a few minutes as Cargo downloads and compiles all dependencies. Subsequent builds will be much faster thanks to incremental compilation.

Frontend Setup

The frontend is a SvelteKit application located in apps/web:

cd apps/web
bun install
bun run dev

The development server will start and open in your browser. By default, the frontend connects to the API at http://127.0.0.1:3001.

Configuring the API URL

If your backend runs on a different host or port, create a .env.local file in apps/web:

# apps/web/.env.local
PUBLIC_API_BASE=http://your-host:your-port

This environment variable is read by SvelteKit at build time and during development to route all API requests to the correct backend.

Project Structure

MCInfo-RS is organized as a Cargo workspace with multiple crates, each handling a specific domain:

info-mc-rs/
  crates/
    mc-protocol/    # Minecraft protocol (SLP, Query, Bedrock)
    mc-mojang/      # Mojang API client (profiles, skins)
    mc-cache/       # Caching layer (moka)
    mc-motd/        # MOTD parsing and rendering
    mc-skin/        # 2D skin rendering
    mc-render3d/    # 3D skin rendering (wgpu)
    mc-api/         # REST API server (Axum)
  apps/
    web/            # SvelteKit frontend
    docs/           # Documentation (you are here)
  docs/             # Internal design documents
